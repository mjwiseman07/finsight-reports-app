# Processing Integrity TSC — Deferred to Phase 42.6 (Q2)

Per planning doc v1.10 Q2: SOC 2 Processing Integrity TSC inclusion is deferred to Phase 42.6 / LOCK-42.6.1 for auditor decision. Founder position: do NOT assume Processing Integrity is in-house at LOCK-42.5.3.

**Rationale:** Advisacor processes financial transaction data. Processing Integrity TSC (PI1.x) requires assertion that processing is COMPLETE, VALID, ACCURATE, TIMELY, and AUTHORIZED. At a two-person shop, the supporting control inventory + sampling depth required to attest Processing Integrity across the financial-automation workload is a meaningful undertaking. The founder position is to engage the auditor at Phase 42.6A and decide jointly whether Processing Integrity scope is:

- (a) Included from issuance — requires control activities + window evidence across processing pipelines
- (b) Deferred to a subsequent SOC 2 Type II issuance after the first window
- (c) Limited to specific processing flows the auditor and founder agree are attestable at first issuance

This document marks Processing Integrity as **DEFERRED**, not OUT-OF-SCOPE. Auditor decision is recorded against this placeholder at Phase 42.6A.

**Decision gate:** LOCK-42.6.1 (engagement) → LOCK-42.6.3 (SOC 2 issuance)
